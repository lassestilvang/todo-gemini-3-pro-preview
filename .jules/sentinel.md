## 2026-11-09 - [Critical] IDOR in Gamification Logic (Revisited)
**Vulnerability:** Although the internal helper `updateUserProgress` explicitly verified user ownership via `requireUser(userId)`, the exported public Server Actions `addXP` and `checkAchievements` in `src/lib/actions/gamification.ts` did not independently validate the session before delegating execution. While protected indirectly, best security practice dictates explicit validation at the entry point to prevent future refactoring regressions.
**Learning:** Publicly exported Server Actions must independently and explicitly validate authorization (e.g., `requireUser(userId)`) at their top level, even if they delegate their entire implementation to another authenticated internal helper. Defense-in-depth is preferable to implicit protection.
**Prevention:** Always place `await requireUser(userId)` as the very first line of any exported Server Action that takes a `userId` argument.
